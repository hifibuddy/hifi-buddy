# v1.2.0 — Release Prep

This file is a release-prep scratchpad. Delete after publishing if you want.

## Order of operations

```bash
# 1. Confirm clean working tree status, then stage everything
cd ~/code/hifi-buddy
git status            # review
git add -A
git diff --cached --stat   # final sanity check

# 2. Commit + tag
git commit -F hifi-buddy-app/docs/V1.2-RELEASE.commitmsg.txt
# (or paste the message below interactively if you prefer)
git tag v1.2.0
git push origin main --tags

# 3. Build the Mac DMG
cd hifi-buddy-app
./build-mac.sh
# Produces: dist/HiFi Buddy.app and dist/HiFi Buddy.dmg

# 4. Test the bundle once
open 'dist/HiFi Buddy.dmg'
# Drag into /Applications, right-click → Open the first time, confirm
# "Open Anyway" in System Settings if needed.

# 5. Publish the GitHub Release with the DMG attached.
#    The "#HiFi-Buddy.dmg" suffix renames the asset on upload so the
#    download URL has no spaces (.../latest/download/HiFi-Buddy.dmg).
gh release create v1.2.0 \
  "dist/HiFi Buddy.dmg#HiFi-Buddy.dmg" \
  --title "v1.2.0 — macOS app + Spotify Connect + Reference Library polish" \
  --notes-file hifi-buddy-app/docs/V1.2-RELEASE.notes.md

# 6. Verify the download URL works:
curl -sIL "https://github.com/hifibuddy/hifi-buddy/releases/latest/download/HiFi-Buddy.dmg" | head -2

# 7. The site update is already in this commit — Vercel auto-deploys
#    from main, so the download buttons go live as the push lands.
```

## Commit message (plain text — copy or use --file)

```
feat(v1.2.0): macOS app, Spotify Connect, Reference Library SDK playback

Sprint 2 part 1: HiFi Buddy is now a real macOS app you can double-click.
Plus a substantial Spotify UX overhaul and a handful of polish + bug fixes.

Distribution
- launcher.py wraps server.py in a menu-bar shell (pystray icon, default-
  browser auto-open, "Reveal config folder", clean shutdown)
- build-mac.sh produces dist/HiFi Buddy.app + dist/HiFi Buddy.dmg
  - Unsigned for v1.x — right-click → Open on first launch
  - LSUIElement = true so it's menu-bar-only, no Dock icon
  - Singleton lock + auto-pick port in 8090..8099 range
- docs/BUILDING.md covers install/sign/run for contributors
- requirements-build.txt: pyinstaller + pystray + Pillow

Spotify
- "Connect to Spotify" button in Settings → Spotify section. Saves
  Client ID + auth method, awaits server config write, then triggers
  PKCE redirect or Client Credentials fetch.
- Reference Library Spotify button now plays the clip via Web Playback
  SDK with seek-to-start, instead of opening Spotify Web search.
- The bottom audio-player bar gains an "external mode" — same chrome,
  controls route through Spotify SDK callbacks. Progress scrubber shows
  position WITHIN the clip segment, not within the full track.
- Pause/resume toggle added to the playline transport (the ▶/⏸ button
  next to the progress bar). Top source buttons stay as source selectors.
- post-render rehydrate: navigating back to a lesson with Spotify still
  playing now restores the visual state (playing button, source-quality
  card, progress tracker).

Brand
- Dashboard hero uses the same 5-bar gradient wave + "HiFi" + gradient
  "Buddy" treatment as the top-bar logo. No more solid-purple title.
- Welcome card (first-run onboarding) gains the same animated soundwave
  used on hifibuddy.net hero — visual continuity.

Fixes
- Default port 8090 (was 8091) — aligns with common Spotify Dev app
  redirect URIs. Updated everywhere.
- ABX decodeAudioData fallback was throwing "Cannot perform slice on
  detached ArrayBuffer" because decodeAudioData detaches its input on
  failure. Now slices upfront so both attempts get fresh buffers, and
  surfaces the real codec/content-type error instead of the masking one.
- Lesson generator: equipment block is now optional with a sensible
  default. Smaller LLMs (Ollama) frequently drop secondary objects;
  failing the whole generation over a missing recommendation tile was
  too strict.
- Player bar height bumped 64 → 92px so the context chip + title +
  artist stack actually fits inside the bar (was overflowing).
- Context chip max-width bumped 220 → 360px for longer reference clip
  labels ("REFERENCE CLIP · MONEY FOR NOTHING" no longer truncates).

Site (hifibuddy.net)
- New "Download or run from source" install section: macOS DMG card
  links to releases/latest/download/HiFi-Buddy.dmg, Windows card shows
  "coming soon" until M4.4 CI lands. Existing git-clone block stays
  as the "or run from source" fallback.

Co-Authored-By: Claude Opus 4.7 (1M context) <noreply@anthropic.com>
```

## GitHub Release notes (Markdown — uploaded via `--notes-file`)

```markdown
HiFi Buddy is now a real macOS app — drag-to-Applications, no Terminal needed.

## Highlights

- 🎉 **Native macOS app** (`HiFi-Buddy.dmg`) — drop into `/Applications`, right-click → Open the first time, then it just works. Menu-bar icon, no Dock clutter, no Python install required.
- 🟢 **Connect to Spotify in Settings** — a real "Connect" button next to your Client ID. PKCE redirect or Client Credentials, your call.
- 🎚 **Reference Library plays Spotify clips in-app** — same player bar as Plex/local clips, with a scrubber that tracks the clip segment (not the whole song).
- 🔁 **Pause/resume on the playline** — the ▶/⏸ button next to the progress bar is now a real toggle, not just a state indicator.
- 🌊 **Brand polish** — dashboard hero, welcome card, top bar all share the same wave + gradient treatment.

## Fixes

- ABX "Cannot perform slice on detached ArrayBuffer" decoder error → real codec error now surfaces instead.
- Lesson generator no longer rejects whole drafts when the AI omits the optional equipment block.
- Default port aligned to **8090** across the codebase — matches the most common Spotify Dev app redirect URI.
- Player bar resized so the "REFERENCE CLIP · …" chip fits inside.

## Install

Easiest: download `HiFi-Buddy.dmg` from the assets below.

> First launch: macOS Gatekeeper will refuse to open an unsigned app. **Right-click → Open**, then confirm "Open Anyway" in *System Settings → Privacy & Security* if prompted. After this one-time confirmation, double-clicking works normally.

Or run from source: `git clone https://github.com/hifibuddy/hifi-buddy && cd hifi-buddy/hifi-buddy-app && python3 server.py`.
```
